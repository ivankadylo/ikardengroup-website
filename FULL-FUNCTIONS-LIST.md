### **B1.1. ОНЛАЙН ПЛАТЕЖІ (COMGATE/GOPAY)**

**ЩО ЦЕ:**
Інтеграція платіжної системи для приймання карток онлайн.

**НАВІЩО:**
- Клієнт платить картою одразу
- Автоматичне підтвердження
- Гроші йдуть на ваш рахунок
- Не треба вручну обробляти

**ЯК ПРАЦЮЄ:**
```
Клієнт додає товар в кошик
→ Натискає "Zaplatit"
→ Перенаправляється на Comgate
→ Вводить дані картки
→ Платить
→ Гроші на ваш рахунок через 2 дні
→ Ви отримуєте сповіщення
→ Відправляєте товар
```

**ЯКІ СЕРВІСИ:**
- **Comgate.cz** (рекомендую)
- Або GoPay.cz
- Або Stripe.com

**ЩО КОНКРЕТНО ТРЕБА ВІД ВАС:**
```
1. Вибрати сервіс (Comgate або GoPay)
2. Зареєструватись на їх сайті
3. Надати документи:
   - IČO (ідентифікаційний номер)
   - DIČ (податковий номер)
   - Виписка з реєстру
   - Банківський рахунок (IBAN)
   - Копія ІД власника
4. Підписати договір
5. Отримати API ключі
6. Надати мені ключі (секретно)
```

**КОМІСІЯ:**
- Comgate: 1.9% + 2 Kč за транзакцію
- GoPay: від 1.9%
- Stripe: 1.5% + 2.5 Kč

**ЧАС ВИКОНАННЯ:** 12-16 годин
**ЧАС НА РЕЄСТРАЦІЮ:** 2-5 робочих днів (схвалення)

**ЧИ РОБИТИ ЗАРАЗ?** ⚠️ ВАШЕ РІШЕННЯ

❌ Все в localStorage браузера
❌ Дані зникають при очищенні
❌ Не синхронізується
❌ Не надійно
❌ Немає історії

**З BACKEND:**
✅ Дані на сервері (надійно)
✅ Історія замовлень
✅ Синхронізація
✅ Можна робити платежі
✅ Можна відправляти email
✅ Можна робити аналітику

**ЯК ПРАЦЮЄ:**
```
Клієнт замовляє товар

→ Зберігаються в базі даних
→ Обробляються
→ Надсилається email
→ Зберігається історія
```

**ЯКІ ВАРІАНТИ:**

**ВАРІАНТ 1: Node.js + MongoDB (рекомендую)**
```
Переваги:
✅ Швидкий
✅ Сучасний
✅ JavaScript (як frontend)
✅ Гнучкий
✅ Безкоштовний хостинг (Render.com, Railway.app)

Мінуси:
❌ Треба вивчити (якщо самі підтримуватиме)
```

**ВАРІАНТ 2: PHP + MySQL**
```
Переваги:
✅ Простіший
✅ Багато хостингів підтримують
✅ Дешевий хостинг
✅ Легко знайти розробника

Мінуси:
❌ Старіший
❌ Повільніший
```

**ВАРІАНТ 3: Firebase (Google)**
```
Переваги:
✅ Не треба свій сервер
✅ Автоматичне масштабування
✅ Безкоштовний план
✅ Швидкий
```
