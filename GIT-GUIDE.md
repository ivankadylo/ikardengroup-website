# 🔄 GIT - ПОВНА ІНСТРУКЦІЯ

## 📚 ЩО ТАКЕ GIT?

Git - це система контролю версій. Вона дозволяє:
```
✅ Зберігати "знімки" проекту в різні моменти часу
✅ Повертатись до будь-якої збереженої версії
✅ Бачити історію всіх змін
✅ Порівнювати версії між собою
```

---

## 🚀 ШВИДКИЙ СТАРТ:

### **ЗБЕРЕГТИ ПОТОЧНУ ВЕРСІЮ:**

#### **Спосіб 1: Через .bat файл (найпростіше)**
```
1. Подвійний клік на: git-commit.bat
2. Все! Версія збережена
```

#### **Спосіб 2: Через командний рядок**
```cmd
cd C:\Users\ivank\muscle-diary\ikarden-website
git add -A
git commit -m "Опис змін"
```

#### **Спосіб 3: Через VS Code**
```
1. Відкрити папку в VS Code
2. Ctrl+Shift+G (Source Control)
3. Натиснути "+" біля файлів
4. Написати повідомлення
5. Натиснути ✓ Commit
```

## 🔄 ВІДНОВИТИ ВЕРСІЮ:

### **Через .bat файл:**
```
1. Подвійний клік на: git-restore.bat
2. Переглянути список версій
3. Скопіювати HASH версії (7 символів)
   Наприклад: a1b2c3d
4. Вставити в поле
5. Натиснути Enter
```

### **Через командний рядок:**
```cmd
cd C:\Users\ivank\muscle-diary\ikarden-website

# 1. Подивитись доступні версії
git log --oneline

# 2. Відновити до конкретної версії
git checkout a1b2c3d .

# 3. Зберегти відновлену версію (опціонально)
git add -A
git commit -m "Restored to version a1b2c3d"
```

### **Повернутись до останньої версії:**
```cmd
git checkout main .
```

## 📝 ВАЖЛИВІ КОМАНДИ:

### **Статус проекту:**
```cmd
git status
```
Показує:
- Які файли змінені
- Які файли готові до збереження
- На якій гілці ви знаходитесь

### **Історія змін:**
```cmd
# Коротка (рекомендую)
git log --oneline

# Детальна
git log

# З графіком
git log --oneline --graph --all

# Останні 5 збережень
git log --oneline -5
```

### **Порівняння версій:**
```cmd
# Що змінилось з моменту останнього збереження
git diff

# Порівняти дві версії
git diff a1b2c3d e4f5g6h
```

### **Інформація про конкретне збереження:**
```cmd
git show a1b2c3d
```

## 💾 ПРОЦЕС ЗБЕРЕЖЕННЯ:

### **1. Перевірити що змінилось:**
```cmd
git status
```

### **2. Додати файли до збереження:**
```cmd
# Всі файли
git add -A

# Конкретний файл
git add index.html

# Всі файли в папці
git add css/
```

### **3. Зберегти з повідомленням:**
```cmd
git commit -m "Опис змін"
```

**Приклади хороших повідомлень:**
```
✅ "Added photo gallery with drag & drop"
✅ "Fixed slider navigation buttons"
✅ "Updated product prices"
✅ "v1.0.0 - Initial release"

❌ "changes"
❌ "fix"
❌ "update"
```

## 🏷️ TAGS (МІТКИ ВЕРСІЙ):

### **Створити мітку:**
```cmd
# Для поточної версії
git tag v1.0.0

# З повідомленням
git tag -a v1.0.0 -m "First stable release"

# Для старішої версії
git tag v0.9.0 a1b2c3d
```

### **Переглянути мітки:**
```cmd
git tag
```

### **Видалити мітку:**
```cmd
git tag -d v1.0.0
```

### **Відновити до мітки:**
```cmd
git checkout v1.0.0 .
```

---

## 🌿 ГІЛКИ (BRANCHES):

### **Переглянути гілки:**
```cmd
git branch
```

### **Створити нову гілку:**
```cmd
# Створити
git branch feature-new-design

# Перейти на неї
git checkout feature-new-design

# АБО обидві команди разом
git checkout -b feature-new-design
```

### **Перейти на іншу гілку:**
```cmd
git checkout main
```

### **З'єднати гілки:**
```cmd
# Перейти на main
git checkout main

# З'єднати з іншою гілкою
git merge feature-new-design
```

## 🔍 ПОШУК:

### **Пошук в історії:**
```cmd
# Знайти commit з словом "gallery"
git log --all --grep="gallery"

# Знайти коли змінювався файл
git log -- index.html

# Хто змінював файл
git blame index.html
```

### **Пошук в коді:**
```cmd
# Знайти слово в усіх файлах
git grep "function"

# В конкретній версії
git grep "function" a1b2c3d
```

## 🛠️ КОРИСНІ ПОРАДИ:

### **1. Часте збереження:**
```
💡 Зберігайте після кожної важливої зміни
💡 Краще 10 маленьких збережень ніж 1 велике
💡 Завжди пишіть зрозумілі повідомлення
```

### **2. Перед великими змінами:**
```
💡 Створіть нову гілку
💡 Тестуйте в ній
💡 Якщо все ОК - з'єднайте з main
```

### **3. Резервні копії:**
```
💡 Git - це не backup!
💡 Регулярно копіюйте папку .git
💡 Використовуйте GitHub/GitLab для онлайн backup
```

### **4. Що НЕ зберігати:**
```
❌ Великі файли (>100MB)
❌ Згенеровані файли
❌ Паролі та ключі
❌ node_modules
❌ .env файли
```

### **git-commit.bat**
```
Швидке збереження поточної версії

### **git-log.bat**
```
Показує історію всіх збережень
Графічний вигляд
```

### **git-restore.bat**
```
Відновлення до певної версії

## 🚨 ВИРІШЕННЯ ПРОБЛЕМ:

### **"Not a git repository"**
```cmd
# Ініціалізувати Git
git init
```

### **"Uncommitted changes"**
```cmd
# Зберегти зміни
git add -A
git commit -m "Save current work"

# АБО скасувати зміни
git reset --hard
```

### **Видалити останнє збереження:**
```cmd
# Скасувати останній commit (зберегти файли)
git reset --soft HEAD~1

# Скасувати останній commit (видалити зміни)
git reset --hard HEAD~1
```

### **Відновити видалений файл:**
```cmd
# З останнього збереження
git checkout HEAD -- index.html

# З конкретної версії
git checkout a1b2c3d -- index.html
```

## 📚 СТРУКТУРА .git ПАПКИ:

```
.git/
├── HEAD              # Вказує на поточну гілку
├── config            # Налаштування проекту
├── description       # Опис проекту
├── hooks/            # Автоматичні скрипти
├── objects/          # Збережені версії файлів
├── refs/             # Посилання на гілки та мітки
└── logs/             # Історія змін
```

**⚠️ НЕ ВИДАЛЯЙТЕ .git ПАПКУ!**
```
Без неї ви втратите всю історію версій!
```

## 🎯 ТИПОВІ СЦЕНАРІЇ:

### **Сценарій 1: Щоденна робота**
```cmd
# Ранок
git status                          # Що змінилось
git add -A                          # Додати зміни
git commit -m "Morning changes"     # Зберегти

# Вечір
git status
git add -A
git commit -m "Evening updates"
```

### **Сценарій 2: Експеримент**
```cmd
# Створити гілку для тесту
git checkout -b test-new-feature

# Працювати, тестувати
git add -A
git commit -m "Testing new feature"

# Якщо все ОК - з'єднати
git checkout main
git merge test-new-feature

# Якщо НЕ ОК - просто видалити гілку
git branch -d test-new-feature
```

### **Сценарій 3: Щось зламалось**
```cmd
# Переглянути історію
git log --oneline

# Знайти останню робочу версію
# Припустимо це a1b2c3d

# Відновити
git checkout a1b2c3d .

# Зберегти як нову версію
git add -A
git commit -m "Restored to working version"
```
